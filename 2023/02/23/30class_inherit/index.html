父类中未定义的方法，直接可以在初始化方法调用？
2023-02-23|python

上面这段代码示例，一共四个类，后面三个都继承了父类Card:

  • 父类中初始化方法调用了_points()方法，但是这个方法在父类中并没有得到实现？–主要是因为这个是初始化方法，且使用中并没有通过Card类去作实例化操作，而都是直接通过子类来进行实例化，这个时候，相当如就是调用子类的这个_points()方法了，而下面每个子类都有实现自己的_points()方法，所以代码能够正常运行的；

  • 子类因为继承了父类，如果子类没有重写__init__方法的话，那在子类实例化的时候，是会自动调用父类的初始化方法的；

  • 如果子类重写了__init__方法，那就会直接使用子类自己的初始化方法；

  • 当然子类也可以通过实现自己的初始化方法，并且同时使用父类初始化方法的一些逻辑,使用 super().__init__(XXX) 时，需要先在子类的初始化方法中定义好需要传递给父类初始化方法的参数，然后再调用 super().__init__() 方法传递这些参数：

    在上面的例子中，Child 类继承了 Parent 类，同时重写了 __init__() 方法，并在其中添加了一个 age 参数。在 Child 类的初始化方法中，首先需要定义好 age 参数，然后调用 super().__init__(name) 方法传递 name 参数给 Parent 类的初始化方法。这样，Child 类就既重写了自己的 __init__() 方法逻辑，同时也继承了 Parent 类的初始化方法逻辑。
