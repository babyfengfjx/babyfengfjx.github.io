什么是鸭子类型

当我们在写代码的时候，如果我们希望一个对象能够执行某个方法，那么我们通常需要先判断这个对象的类型是否是我们所需要的类型。但是在鸭子类型中，我们不需要知道这个对象的类型，只需要关心它是否具有我们所需要的方法或属性即可。

换句话说，鸭子类型中，我们不关心对象的类别，只关心对象是否能执行我们需要的方法或属性。这就好像看到一只鸟，我们并不需要知道它的种类是否是鸭子，只要它具有“走路、游泳、叫声”等鸭子的特征，我们就可以把它看作是一只鸭子。

在 Python 中，鸭子类型可以让代码更加简洁、灵活，同时也可以提高代码的可复用性和通用性。当我们使用鸭子类型时，我们不需要对对象的类型进行过多的考虑，只需要关注对象是否具有所需的方法或属性，这样可以让我们的代码更加通用、灵活，也更容易扩展和维护。

在 Python 中，鸭子类型指的是一种动态类型检查方式，即只要对象的方法或属性能够满足所需的功能，它就可以被看作是所需的类型，不需要进行显式的类型检查。这意味着，如果一个对象具有与特定类型相同的方法和属性，那么它就可以被视为是该类型。

一句话理解就是：某个对象只要有我们要的方法和属性我们就可以拿过来用，不需要判断这个对象是什么类型，是否符合我们对具体对象的限制，有就拿来用就是。

类中方法分类

  1. 实例方法（Instance Methods）：实例方法是指在类中定义的函数，它需要至少一个参数 self，用于访问实例的属性和方法。实例方法可以直接通过实例调用，例如 my_instance.my_instance_method()。
  2. 静态方法（Static Methods）：静态方法是指在类中定义的函数，它不需要访问类或实例的属性或方法。静态方法通常用于执行某种通用功能，与类和实例无关。静态方法可以直接通过类调用，例如 MyClass.my_static_method()。
  3. 类方法（Class Methods）：类方法是指在类中定义的函数，它需要一个参数 cls，用于访问类的属性和方法。类方法通常用于执行某种与类相关的操作。类方法可以通过类直接调用，例如 MyClass.my_class_method()。
  4. 魔术方法（Magic Methods）：魔术方法也称为特殊方法（Special Methods），是 Python 中用于实现类的特殊行为的方法。例如 __init__ 用于初始化对象，在创建对象时自动调用；__str__ 用于返回对象的字符串表示等。魔术方法的调用通常是由 Python 解释器自动触发的。
  5. 属性方法（Property Methods）：属性方法是一种特殊的实例方法，用于对类的属性进行访问和设置。它们可以像属性一样直接访问和设置，而不需要使用方法调用的语法。属性方法通常用于隐藏属性的实现细节，并在属性访问时执行某些计算。

主要区别

这些方法的不同之处在于参数、作用范围和调用方式等。

  • 实例方法是最常见的方法类型，它是定义在类中的一个函数，通过实例化类对象后可以直接调用。实例方法的第一个参数一般是 self，表示实例本身。实例方法可以访问实例属性和实例方法，也可以访问类属性和类方法。
  • 静态方法通过 @staticmethod 装饰器定义，它不需要第一个参数表示实例本身，因此在定义时不需要 self 参数。静态方法可以通过类名或实例名直接调用，不依赖于类实例。通常情况下，静态方法是一些不需要访问实例属性或者类属性的方法。
  • 类方法通过 @classmethod 装饰器定义，第一个参数一般是 cls，表示类本身。类方法可以通过类名或实例名调用。类方法可以访问类属性和类方法，不能访问实例属性和实例方法。
  • 属性方法通过 @property 装饰器定义，它是一个特殊的方法，可以把方法访问方式变成属性访问方式，可以像访问属性一样访问方法。属性方法通常用于访问和设置实例属性，而不需要使用getter或setter方法。属性方法不需要传递参数，通过访问器getter和修改器setter方法进行操作。

使用场景

  • 实例方法通常用于处理实例的属性或状态，并且需要访问实例的属性或调用实例的方法。它们是最常见的方法类型，并且可以访问实例的任何属性和方法。
  • 静态方法通常用于不需要访问实例或类属性的辅助函数。它们可以直接调用，无需创建实例或访问类。
  • 类方法通常用于需要访问类属性或操作类级别数据的函数。它们可以使用类属性，而不需要访问实例属性。它们通常用于创建工厂方法或在子类中进行多态处理。
  • 属性方法通常用于将一个方法像一个属性一样访问，并在访问它时动态计算结果。它们可以用于对实例属性进行操作，而不需要直接暴露属性本身。

总结

  • 类方法是可以直接使用类变量的，直接通过cls. 的方法即可使用（cls指类本身）；
  • 实例方法也是可以使用类变量的，可以通过类名. 的方式，或者直接通过 self.class_variable 方式调用；–实例访问类变量时，其实是通过实例的 __class__ 属性来获取实例对应的类，然后再通过类来访问类变量。虽然可以这样做，但是这样会让代码变得混乱不清，不建议这样使用；
  • 静态方法不能直接访问实例变量，因为它们不接收 self 参数。静态方法的主要作用是为了将某些操作与类关联起来，并且与实例无关，因此通常情况下，静态方法只与类的静态属性有关。如果静态方法需要访问实例变量，可以将实例变量作为参数传递给静态方法，从而实现访问实例变量的目的;
  • 类方法默认是无法访问实例变量的，因为类方法没有默认的 self 参数，无法获取到实例的引用。但是如果你想要在类方法中访问实例变量，你可以手动传入一个实例作为参数，或者使用 classmethod 装饰器将类方法转换为一个“类和实例”方法
