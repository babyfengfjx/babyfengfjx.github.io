多线程：threading

threading是Python中内置的线程库，通过使用该库，我们可以在同一个进程中创建多个线程，并发执行多个子任务，从而提高程序的运行效率。

thread类

Thread类是Python threading库中最基础的类。如果我们要使用多线程编程，那么必须先创建Thread对象，然后将任务放入Thread对象中进行处理。

Thread类有以下常用的方法：

  • run()方法：表示线程要执行的代码，我们需要在该方法中编写线程的逻辑。
  • start()方法：表示启动线程，程序会自动调用run()方法来执行线程逻辑。
  • join()方法：表示等待线程结束，程序会一直等待线程执行完毕后才会继续向下执行。

Lock类

多个线程在同一时间对共享资源进行读写操作时，可能会引发数据不一致等问题，为了避免这些问题，我们可以使用Lock类对共享资源进行加锁。

Lock类有以下常用的方法：

  • acquire()方法：表示获取锁，如果锁已经被其他线程获取，则当前线程会一直等待。
  • release()方法：表示释放锁。

下面来详细讲解一下 threading 库。

threading 库是 Python 中用于创建和管理线程的标准库，其主要包括以下模块和类：

  • threading.Thread：线程类，用于创建和管理线程
  • threading.Lock：锁类，用于控制线程之间的访问和修改共享资源的顺序
  • threading.RLock：可重入锁类，与锁类类似，但允许同一线程多次获取锁
  • threading.Condition：条件变量类，用于线程之间的通信和同步
  • threading.Semaphore：信号量类，用于控制并发访问的数量
  • threading.Event：事件类，用于线程之间的通信和同步
  • threading.Timer：定时器类，用于在指定时间后执行特定操作
  • threading.current_thread()：函数，返回当前线程对象
  • threading.active_count()：函数，返回当前活动的线程数
  • threading.enumerate()：函数，返回当前活动的线程列表

对于高并发场景，使用多进程和异步通常会比使用多线程更加高效。多进程可以充分利用多核CPU的性能，并且不会受到GIL的限制。异步编程则可以减少线程的切换和上下文切换开销，提高并发能力。

不过在选择使用哪种并发编程方式时，需要考虑到具体的场景和需求，包括任务的性质、数据的规模、系统的资源等等因素。同时，不同的并发编程方式也有各自的优缺点和适用范围，需要进行权衡和选择。

例：多线程下载文件

生产中有很多应用需要验证，我们需要先下载到本地安装，所以下载应用安装包是可以利用多线程来完成的。

threading 模块是 Python 中用于创建和管理多线程编程的内置模块。它提供了许多类和方法，使得创建和管理线程变得更加容易和高效。

Python 中的 threading 模块中最常用的类有：

  • Thread: 表示一个线程的类。
  • Lock: 可以用于线程间共享的锁。
  • RLock: 可以重入的锁，允许一个线程多次请求相同的锁。
  • Semaphore: 表示可以同时运行的线程数量的类。
  • Condition: 用于线程之间的通信。

除此之外，还有一些其他的类，如Timer、Event、Barrier等，在不同场景下可以选择合适的类来实现需要的功能。

在 threading 模块中，最常用的方法是 Thread() 函数，使用该函数可以创建一个新的线程。

ThreadPoolExecutor：

  • ThreadPoolExecutor是Python标准库concurrent.futures中的一个类，它允许我们在Python中使用线程池的概念来执行多个任务，并且可以限制线程的数量。这是一个非常实用的类，特别是当你需要执行很多短暂的任务时，可以充分利用CPU资源，提高性能。

  • ThreadPoolExecutor的用法十分简单，它只需要两个参数：要运行的任务和线程池的大小。使用submit方法将任务提交给线程池。这个方法会返回一个Future对象，代表这个任务的未来的结果。

一般来说，我们可以通过as_completed函数来获取已经完成的任务的结果。as_completed会在每个任务完成后返回这个任务的结果，并且返回这些结果的顺序是无保证的。

还可以通过map函数来获取任务的结果，map函数将返回一个迭代器，这个迭代器可以按照任务提交的顺序，返回每个任务的结果。

add_done_callback:

在 threading 模块中，Thread 对象并没有提供 add_done_callback() 方法来注册回调函数，因为 threading.Thread 类是简单的线程类，不同于 asyncio.Task 对象，其并没有设计来执行异步任务并支持注册回调函数。

但是，我们可以使用 Python 内置的 concurrent.futures 模块来实现类似于 asyncio.Task 对象中的 add_done_callback() 方法的功能。具体的做法是，首先使用 concurrent.futures.ProcessPoolExecutor 或 concurrent.futures.ThreadPoolExecutor 创建一个线程池或进程池，然后将任务提交给线程池或进程池处理，使用 submit() 方法返回的 Future 对象支持 add_done_callback() 方法，可以注册回调函数，等待任务完成后自动调用回调函数。

在 future.add_done_callback(callback_func) 中，callback_func 方法不需要传递参数。因为 add_done_callback() 方法会自动将 Future 对象作为参数传递给回调函数，所以回调函数只需要一个参数来接收 Future 对象即可。可以在回调函数中使用 future.result() 方法来获取任务的结果。

具体来说，当任务完成时，Future 对象会自动调用回调函数，并将该对象作为唯一的参数传递给回调函数。回调函数可以通过 future.result() 方法获取任务的结果，或者通过 Future 对象的其他方法获取任务的状态和元数据。回调函数不需要显式地传递任何参数，因为 Future 对象已经封装了所有相关的信息。
